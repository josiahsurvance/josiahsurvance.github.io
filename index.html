Parkour Infinite (for CGB)

This gamemode (there are two gametypes) uses checkpoints, teleporters, and more to create a unique parkour experience. The instructions for how to use the tools to create your own map are below. This section will explain how some of the things work while in the actual game. Also, you NEED the regular Parkour Infinite gametype for everything to work correctly in custom games or the CGB.

Four Weapon System: Players have access to 4 weapons throughout the game. The first two weapons, along with their corresponding abilities, can be customized/changed by the map creator using Weap/Abil Zones objects, detailed below. The third weapon is the shotgun, which has a jetpack tied to it. This jetpack allows the player to access Practice Mode, detailed below. The fourth weapon is the DMR, which has a drop shield tied to it. While holding this and using drop shield, it allows players to teleport between unlocked checkpoints. I wanted players/speedrunners to be able to quickly reset or practice any section at any time, while still being able to use the weapons/abilities the map creator wanted; this four weapon system was my solution. This can be turned off using an appropriately labeled Info Zone, which will cause the regular two weapon system to be used.

Practice Mode: When the player uses a jetpack with a shotgun, whether its their third weapon or not, they will enter Practice Mode. This pauses their timers and deaths. While in this mode, if they walk over any checkpoint, this will become the default checkpoint they teleport to when in a Teleporter object zone. This is meant to allow players to pause their timer, practice a specific jump in a level, or explore the rest of the map.

To leave practice mode, the player simply needs to go back to their checkpoint and wait for 5 seconds, with an accompanying progress bar. The players timer and death counter resumes, and the weapons and abilities they had before entering Practice Mode are reassigned (grenades and any vehicle they have are deleted). It is noted that if someone accidentally enters practice mode, they can still “beat” the map, as teleporters will update based on checkpoints they reach; they just won’t legitimately beat the map with the timer and scoreboard. I say this to say that you can still “complete” the map in this mode, meaning you don’t have to worry about noobs being stuck in practice mode and unable to progress.

Also, you may add a Teleporter object with spawn sequence 6 that acts as a zone in which players may not use Practice Mode. So, if you want to prevent players from accessing end-map weapons and vehicles to troll with, use this zone. You can also use these zones in between checkpoints if you would like for players to be able to practice the specific checkpoint they are on, but no others.

Special Armor Abilities:

  • Drop Shield:

    • Activating drop shield normally puts player into Spectator Mode

    • If player is holding a DMR, activating drop shield allows player to teleport between unlocked checkpoints.

  • Active Camo:

    • If player has been given blocks from Give Object zone, activating active camo will place down one of the allotted blocks with each ability use.

  • Jetpack:

    • If player is holding shotgun, player will enter Practice Mode when used.

      Spectator Mode: In this state, the player is placed into a monitor. While in this monitor, their timers and deaths are paused, similar to Practice Mode. These monitors can only spectate; they cannot progress to further checkpoints. They are also unable to tamper with other players; if a monitor comes too close to another player, they immediately get ejected from Spectator Mode. You can also see other players’ stats above their head while in Spectator Mode. Players can leave Spectator Mode by using their armor ability (active camo).

      Certain objects work only on CGB as opposed to customs (for the host): Moving Block and Rotating Object objects only work on CGB for everyone. They will still work in private customs for everyone except the host; just keep this in mind. This was done for major optimization purposes and allows for more functionality. The active camo block placing ability might be added to this list potentially, but is currently not.


      Miscellaneous: Player collision is off by default and the Four Weapon System is on by default. These can be toggled by creating the appropriate Info Zone/Misc objects if desired. Just uh, at the very least, please leave Practice Mode on if you’re making a hard map; it helps a lot.

      By the same method you can use the appropriate Info Zone/Misc object to enable vehicle parkour. When this is toggled on, Practice Mode is replaced by the ability to spawn the vehicle of your choice. My “Joe’s Modded Mongoose Parkour” utilizes this.

      Script created vehicles are invincible, meaning they can be outside the map/underwater. Forge created vehicles are regular health. All Scale objects are invincible, so you can give them this label if you want them invincible as well.

      Speaking of invincibility, players are fully invincible by default (can be turned off with Trait Zone). This means players can also be outside the map/underwater and not die. PC players can now also build outside the map and then use these maps on the CGB. See this video.

      Parkour Infinite Editing (for Forge)

      This is the second of the two gametypes. To use all the fancy functions within the forge variant, an object must first die. I recommend that you kill your player to achieve this. Wherever you die, a kill ball will be spawned over your head. This kill ball is what makes the forge scripts work. Rest assured, this does not happen in the CGB variant. It’s just necessary for forge. Once the kill ball is spawned, it should be repeatedly killing a monitor; this means it’s working. To turn off the fancy effects/reset, simply start a new round.

      You spawn in with the traits that players have in the actual game. Standing in the multiple zones will allow for normal functionality, although the special abilities have been modified (teleport to checkpoints) or disabled (Spectator Mode) for the forge mode. Walking into Trait Zones and Weap/Abil Zones, for example, affects your player in the same way as in an actual game. Also, walking into a race_flag zone will make this the default object that Teleporter objects will send you to if the object is configured correctly. This is done for testing purposes in the forge mode. You can also use bubble shield to teleport between all the checkpoints on the map, which also makes it the default Teleporter object.

      If something appears to not be working, start a new round and try again. Restarting a round almost always fixes the issues. Also, playing in forge with multiple players creates other bugs as well for non-host players, just FYI. This is just in the Forge mode, rest assured.

      IMPORTANT TIP: When setting up labels, I recommend leaving the spawn sequence and team for last. Leave the spawn sequence and team at 0 and none and pick which label you need first. Now adjust the spawn sequence and team accordingly. This should hopefully prevent objects accidentally disappearing or acting strange. Otherwise you might need to restart to fix the object.

    race_flag

    These are your checkpoints. Make sure it has a shape the player can access. Should be intuitive. I made it so it auto sorts them, so you don’t have to worry about having it as “0, 1, 2, 3, etc.” compared to “-5, 2, 3, 7, 11, etc.” Don’t place down two objects with same spawn sequence; the map will still technically work, but one object will always be prioritized over the other. I would recommend starting with 0 and going up though, as waypoint (in forge) shows spawn sequence. Also, definitely start with 0 and go up if you’re using End Teleporter objects. Shapes/waypoints behave as normal race checkpoints, except for “Start Zone,” which has both always visible. Waypoints default as visible in forge for ease of use. Standing in checkpoint will make the checkpoint the default object that you teleport to when you fall into a Teleporter zone that is configured properly.

  • Subsequent spawn sequences go in ascending order

      Despawn Object

      Objects with this label immediately despawn in the correctly configured Teleporter zones (spawn sequence 0 and 4). Shapes/waypoints are always invisible for these objects in-game.     Give Objects

    This zone allows you to give players a certain number of blocks, equal to the spawn sequence, to place using the active camo ability. Functionality works in forge if you have active camo. Waypoint text is spawn sequence number, and defaults as visible in forge. Given blocks resets to zero when the player enters a checkpoint.

    Object Timer

    This allows you to have timed block sections. Setting a blocks spawn sequence is the same as setting its timer. And then simply make at least one object with a zone with spawn sequence -1 to start all the timers for each block within the corresponding team. In Forge, the objects flicker. This just happens in forge. Basically, if you step in a zone with spawn sequence -1, all +# spawn sequence objects (with the same team) will appear. After leaving zone, all +# spawn sequence blocks will disappear after their timer (set by spawn sequence, in seconds) ticks down.

    You can also just have timed sections that teleport you back to checkpoint if you run out of time. Use neutral teams and object shapes for this. In forge, it will just send you messages updating you on the time you have left. In game, there will be a progress bar. Because of scripting limitations, I can only set a 10 second progress bar. This means, that if you set a timer as 5 seconds, it will still work; it will just be halfway down already. Likewise if you set a time as 15 seconds, it will still work; it will just look like nothings happening for the first 5 seconds. Just keep this in mind. Also, ensure that your zones aren’t overlapping, as this can cause issues.

    Team

  • Neutral: Use this for player timed zones. Players must stand in the object’s shape for the timers to be set and to be run.

    • Spawn Sequence

      • -#’s: Match this to the +# spawn sequence object to make sure everything works properly. These are the zones that cause the timer to go down.

         +#’s: This is the timer you’re setting, corresponding to seconds, that is given to the player.

 +#’s: This is the timer for each block, corresponding to seconds.

Moving Block

There are three ways to create the actual moving blocks: one, hill markers with the Moving Block label and negative spawn sequences will spawn either one or multiple objects of your choice; two, objects with the Despawn Object label can be moving blocks; and three, objects with Scale label can be moving blocks. Note that the objects created by the hill marker with the Moving Block label will have the Despawn Object label, meaning that if the blocks run into a Teleporter Object with spawn sequence 0 or 4, it will despawn and then respawn back at either the hill marker it came from or its original position.

The above objects move because they have the Despawn Object label, meaning that any object you give this label is also able to moved. Also, most blocks that the player creates with the Despawn Object label will despawn by themselves after about 10 seconds. THIS ONLY HAPPENS IN FORGE. They don’t despawn in CGB, so build accordingly.

To make these objects move/rotate, you need to create a block mover. These are made by creating a Moving Block object with a positive spawn sequence and a shape. Now, when the above objects are in this shape, they will move/rotate in the direction that the Moving Block object is rotated/facing and optionally copy its rotation as well, with the spawn sequence controlling speed. You can also solely change the object’s rotation with another Moving Block object.

Again, moving blocks will despawn when hitting a Teleporter Object with spawn sequence 0 or

4. This is the intended way to delete these blocks; otherwise, they might not despawn/respawn…

Its recommended to use the Underjump Toggle with these objects to ensure players fall when they are meant to fall.

    Scale

    Scales objects up or down. Also, if a player stands in a Scale zone, the player will be scaled accordingly. Note that physical hitboxes don’t change with scaling; just the visual hitbox changes. This is the same label (mostly) that is used in Anvil Editor, so scaled objects will work in both of these modes. -10 is the smallest size, -11 is the largest.
